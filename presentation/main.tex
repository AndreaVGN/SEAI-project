          \item $-100$ for crash or boundary exit
          \item $-0.3$ per engine firing (fuel cost)
        \end{itemize}

        \begin{itemize}
          \item \textbf{Initial conditions}: position and velocity
                sampled from a bounded uniform distribution
          \item \textbf{Optional wind}: $\mathcal{N}(0, \sigma_w^2)$
                lateral force added each step
          \item \textbf{Turbulence}: random torque
                $\mathcal{N}(0, \sigma_T^2)$
        \end{itemize}
      \end{block}
      \vspace{0.4em}
      \begin{alertblock}{Reward hacking risk}
        Agent may learn to \emph{hover} without landing to avoid the
        crash penalty. Monitored via episode length and leg-contact
        rate during evaluation.
      \end{alertblock}
  \end{columns}
\end{frame}

% ─────────────────────────────────────────────────────────────────────────────
%  SLIDE 6 — SARSA: THEORY
% ─────────────────────────────────────────────────────────────────────────────
\begin{frame}{Algorithm 1: SARSA — Theory}
  \begin{columns}[T]
    \column{0.50\textwidth}
      \textbf{SARSA} (Sutton \& Barto, 2018, §6.4) is a model-free,
      \keyword{on-policy} TD control algorithm.\\[0.5em]

      At each step the agent uses the tuple
      $(s_t, a_t, r_t, s_{t+1}, a_{t+1})$ to update:

      \[
        \boxed{Q(s_t, a_t) \leftarrow Q(s_t, a_t) +
               \alpha \bigl[r_t + \gamma Q(s_{t+1}, a_{t+1}) -
               Q(s_t, a_t)\bigr]}
      \]

      \begin{itemize}
        \item $a_{t+1}$ is sampled from the \textbf{current policy}
              (typically $\varepsilon$-greedy)
        \item This makes the method \textbf{on-policy}: the target
              includes the next action actually taken — unlike Q-Learning
        \item Converges to the optimal policy under mild conditions
      \end{itemize}
    \column{0.46\textwidth}
      \begin{block}{SARSA vs Q-Learning}
        \vspace{0.3em}
        \begin{tabular}{lcc}
          \toprule
          & \textbf{SARSA} & \textbf{Q-Learning}\\
          \midrule
          Target & $Q(s', a')$ & $\max_{a'}Q(s',a')$\\
          Policy & on-policy & off-policy\\
          Safety & safer & riskier\\
          \bottomrule
        \end{tabular}
      \end{block}
      \vspace{0.5em}
      \begin{block}{Key intuition}
        SARSA is \emph{pessimistic}: it evaluates the policy it
        actually follows, including exploration mistakes.  This
        yields more \textbf{conservative} policies in stochastic
        environments.
      \end{block}
  \end{columns}
\end{frame}

% ─────────────────────────────────────────────────────────────────────────────
%  SLIDE 7 — DEEP SARSA
% ─────────────────────────────────────────────────────────────────────────────
\begin{frame}{Deep SARSA: Neural Network Extension}
  \begin{columns}[T]
    \column{0.50\textwidth}
      Tabular SARSA cannot handle the continuous state space of
      LunarLander-v3 ($\mathcal{S} \subset \mathbb{R}^8$).\\[0.4em]

      We replace the Q-table with a neural network
      $Q_\theta(s, \cdot)$: one forward pass yields Q-values for
      \emph{all} 4 actions simultaneously.\\[0.4em]

      \textbf{Loss function:}
      \[
        \mathcal{L}(\theta) =
        \bigl(Q_\theta(s,a) -
        \underbrace{[r + \gamma Q_{\theta^-}(s', a')]}_{\text{SARSA target}}\bigr)^2
      \]
      where $\theta^-$ are the \keyword{target network} parameters
      (lagged copy, synced every $N$ episodes).

    \column{0.46\textwidth}
      \begin{block}{Extensions over tabular SARSA}
        \begin{itemize}
          \item \textbf{Target network}: stabilises the Bellman
                target, preventing oscillations (Mnih et al., 2015)
          \item \textbf{Short-horizon replay buffer}: de-correlates
                consecutive gradient updates while staying
                \emph{approximately} on-policy
          \item \textbf{Gradient clipping} ($\|\nabla\|_\infty \leq 10$)
                for stability in early training
        \end{itemize}
      \end{block}
      \vspace{0.3em}
      \begin{alertblock}{On-policy integrity}
        Each stored transition also saves $a'$ (the next action from
        the \emph{current} policy), preserving the SARSA semantics
        inside the replay buffer.
      \end{alertblock}
  \end{columns}
\end{frame}

% ─────────────────────────────────────────────────────────────────────────────
%  SLIDE 8 — A2C: THEORY
% ─────────────────────────────────────────────────────────────────────────────
\begin{frame}{Algorithm 2: Advantage Actor-Critic (A2C) — Theory}
  \begin{columns}[T]
    \column{0.50\textwidth}
      A2C is a synchronous \keyword{policy-gradient} method
      (Mnih et al., 2016) combining:

      \begin{itemize}
        \item \textbf{Actor} $\pi_\theta(a|s)$: stochastic policy
        \item \textbf{Critic} $V_w(s)$: state-value baseline
      \end{itemize}

      \vspace{0.4em}
      \textbf{Actor gradient} (REINFORCE with baseline):
      \[
        \nabla_\theta J = \mathbb{E}\bigl[
          \nabla_\theta \log \pi_\theta(a|s) \cdot A(s,a)
        \bigr]
      \]

      \textbf{Advantage} ($n$-step return):
      \[
        A_t = \sum_{k=0}^{n-1}\gamma^k r_{t+k} +
              \gamma^n V_w(s_{t+n}) - V_w(s_t)
      \]

      \textbf{Critic loss}:
      $\mathcal{L}_c = \bigl(G_t - V_w(s_t)\bigr)^2$

    \column{0.46\textwidth}
      \begin{block}{Entropy regularisation}
        \[
          \mathcal{L}_\pi = -\log \pi_\theta(a|s) \cdot A_t
                            - \beta H[\pi_\theta(\cdot|s)]
        \]
        The entropy bonus $\beta H[\pi]$ prevents premature
        convergence to deterministic (brittle) policies.
      \end{block}
      \vspace{0.4em}
      \begin{block}{Why A2C over plain REINFORCE?}
        \begin{itemize}
          \item Critic reduces gradient variance via the baseline
          \item $n$-step returns balance bias–variance trade-off
          \item No replay buffer needed: fully on-policy
        \end{itemize}
      \end{block}
  \end{columns}
\end{frame}

% ─────────────────────────────────────────────────────────────────────────────
%  SLIDE 9 — ALGORITHM COMPARISON
% ─────────────────────────────────────────────────────────────────────────────
\begin{frame}{Algorithm Comparison}
  \begin{center}
  \renewcommand{\arraystretch}{1.35}
  \begin{tabular}{lccc}
    \toprule
    \textbf{Property} & \textbf{Deep SARSA} & \textbf{A2C} \\
    \midrule
    Paradigm       & Value-based (TD)   & Policy gradient \\
    Policy type    & Implicit ($\varepsilon$-greedy on $Q$) & Explicit ($\pi_\theta$) \\
    On/Off policy  & On-policy          & On-policy \\
    Exploration    & $\varepsilon$-greedy & Entropy bonus \\
    Replay buffer  & Yes (short-horizon) & No \\
    Target network & Yes                & No \\
    Networks       & 1 Q-network + target & Actor + Critic \\
    Update freq.   & Per mini-batch     & Every $n$ steps \\
    Convergence    & Q-function → optimal policy & Policy gradient ascent \\
    Sample eff.    & Moderate           & Lower (fully on-policy) \\
    \bottomrule
  \end{tabular}
  \end{center}
  \vspace{0.4em}
  \begin{columns}
    \column{0.48\textwidth}
      \begin{block}{\color{white}SARSA advantage}
        Value-based updates tend to be more \emph{stable} and
        \emph{sample efficient} for discrete-action problems.
      \end{block}
    \column{0.48\textwidth}
      \begin{block}{\color{white}A2C advantage}
        Direct policy optimisation enables smoother convergence and
        \emph{natural entropy control} via $\beta H[\pi]$.
      \end{block}
  \end{columns}
\end{frame}

% ─────────────────────────────────────────────────────────────────────────────
%  SLIDE 10 — NETWORK ARCHITECTURE
% ─────────────────────────────────────────────────────────────────────────────
\begin{frame}{Neural Network Architecture}
  \begin{columns}[T]
    \column{0.50\textwidth}
      \textbf{Deep SARSA} — single Q-network (+ target):
      \[
        \mathbb{R}^8 \xrightarrow{\text{FC-256}} \text{ReLU}
        \xrightarrow{\text{FC-256}} \text{ReLU}
        \xrightarrow{\text{FC-4}} \mathbb{R}^4
      \]
      Output: $Q(s, a)$ for each of the 4 actions.\\[0.8em]
      \textbf{A2C} — two separate networks:
      \[
        \underbrace{\mathbb{R}^8 \!\to\! 256 \!\to\! 256 \!\to\! 4}_{\text{Actor}: \text{Softmax} \Rightarrow \pi_\theta(a|s)}
        \quad
        \underbrace{\mathbb{R}^8 \!\to\! 256 \!\to\! 256 \!\to\! 1}_{\text{Critic}: V_w(s)}
      \]
    \column{0.46\textwidth}
      \begin{block}{Design choices}
        \begin{itemize}
          \item \textbf{Width 256}: sufficient capacity for the 8-D
                state without overfitting
          \item \textbf{ReLU}: avoids vanishing gradients; standard
                choice for control tasks
          \item \textbf{Separate actor/critic}: allows independent
                learning rates ($\alpha_\pi < \alpha_V$), reducing
                the risk of the critic destabilising the actor
          \item \textbf{Orthogonal init} (actor): small initial
                logits for near-uniform initial policy
          \item \textbf{Kaiming init} (Q-network): optimal for ReLU
        \end{itemize}
      \end{block}
  \end{columns}
\end{frame}

% ─────────────────────────────────────────────────────────────────────────────
%  SLIDE 12 — EXPLORATION
% ─────────────────────────────────────────────────────────────────────────────
\begin{frame}{Exploration–Exploitation Trade-off}
  \begin{columns}[T]
    \column{0.50\textwidth}
      \begin{block}{Deep SARSA — $\varepsilon$-greedy}
        \[
          \pi(a|s) = \begin{cases}
            \text{Uniform}(\mathcal{A}) & \text{w.p. } \varepsilon_t\\
            \arg\max_a Q_\theta(s,a)    & \text{w.p. } 1-\varepsilon_t
          \end{cases}
        \]
        $\varepsilon_t = \max(\varepsilon_\infty,\ \varepsilon_0 \cdot
        \lambda^t)$ with $\lambda = 0.995$.\\[0.4em]
        \textbf{Pros}: simple, well-understood decay schedule.\\
        \textbf{Cons}: exploration is \emph{undirected} (uniform random).
      \end{block}
    \column{0.46\textwidth}
      \begin{block}{A2C — Entropy Regularisation}
        \[
          H[\pi] = -\sum_a \pi_\theta(a|s)\log\pi_\theta(a|s)
        \]
        Maximising $\beta H[\pi]$ \emph{discourages} the actor from
        collapsing to a deterministic policy prematurely.\\[0.4em]
        \textbf{Pros}: exploration \emph{adapts} to the policy shape;
        no need to anneal a separate parameter.\\
        \textbf{Cons}: $\beta$ must be tuned; too large $\Rightarrow$
        noisy actions; too small $\Rightarrow$ premature convergence.
      \end{block}
  \end{columns}
  \vspace{0.5em}
  \begin{center}
    \highlight{Key insight}: A2C's entropy exploration is \emph{structured}
    (proportional to policy uncertainty), while SARSA's $\varepsilon$-greedy is
    \emph{uninformed} — relevant when comparing sample efficiency.
  \end{center}
\end{frame}

\begin{frame}{Statistical Analysis}
  \begin{columns}[T]
    \column{0.50\textwidth}
      \textbf{Method}: Welch's $t$-test on mean return over the last
      100 episodes, averaged over 5 seeds.\\[0.5em]

      \textbf{Hypotheses}:
      \begin{itemize}
        \item $H_0$: $\mu_\text{SARSA} = \mu_\text{A2C}$
        \item $H_1$: $\mu_\text{SARSA} \neq \mu_\text{A2C}$
      \end{itemize}
      \vspace{0.5em}

      \renewcommand{\arraystretch}{1.25}
      \begin{tabular}{lcc}
        \toprule
        \textbf{Metric} & \textbf{SARSA} & \textbf{A2C}\\
        \midrule
        Mean return    & $169.83$        & $-213.22$       \\
        Std (seeds)    & $17.49$         & $14.90$         \\
        95\% CI        & $[154.5,\;185.2]$ & $[-226.3,\;-200.2]$ \\
        \midrule
        $t$-statistic  & \multicolumn{2}{c}{$37.27$}      \\
        $p$-value      & \multicolumn{2}{c}{$<0.001$}     \\
        \textbf{Verdict} & \multicolumn{2}{c}{\textcolor{successgreen}{\textbf{Reject $H_0$}}} \\
        \bottomrule
      \end{tabular}
      \vspace{0.3em}\\
      \small\color{successgreen}SARSA significantly outperforms A2C ($\alpha=0.05$)
    \column{0.46\textwidth}
      \begin{block}{Why Welch's $t$-test?}
        \begin{itemize}
          \item Handles \textbf{unequal variances} between the two
                agents (Welch, 1947) — more appropriate than Student's
                $t$-test when $\sigma_S \neq \sigma_A$
          \item With $n=5$ seeds, we report the $p$-value at
                $\alpha = 0.05$
        \end{itemize}
      \end{block}
      \vspace{0.4em}
      \begin{block}{Multiple seed rationale}
        RL results are \textbf{highly seed-dependent} (Henderson
        et al., 2018). Single-seed comparisons are unreliable.
        5 seeds provide statistically meaningful confidence
        intervals.
      \end{block}
  \end{columns}
\end{frame}

% ─────────────────────────────────────────────────────────────────────────────
%  SLIDE 17 — FAILURE ANALYSIS
% ─────────────────────────────────────────────────────────────────────────────
\begin{frame}{Failure Analysis \& Limitations}
  \begin{columns}[T]
    \column{0.50\textwidth}
      \begin{block}{Deep SARSA failure modes}
        \begin{itemize}
          \item \textbf{Hovering}: agent fires main engine to avoid
                the crash penalty without committing to a landing
          \item \textbf{High variance across seeds}: on-policy updates
                with short buffers are sensitive to initial trajectory
                distribution
          \item \textbf{Catastrophic forgetting}: rare when replay
                buffer is too small; policy updates can override
                previously learned behaviours
        \end{itemize}
      \end{block}
    \column{0.46\textwidth}
      \begin{block}{A2C failure modes}
        \begin{itemize}
          \item \textbf{Policy collapse}: if $\beta$ is too small,
                entropy vanishes and the actor gets stuck in a
                sub-optimal deterministic policy
          \item \textbf{Critic lag}: the advantage signal is noisy
                until the critic has seen enough states
          \item \textbf{Sensitivity to learning rates}: separate
                $\alpha_\pi$, $\alpha_V$ require careful tuning;
                mismatch causes instability
        \end{itemize}
      \end{block}
  \end{columns}
  \vspace{0.4em}
  \begin{alertblock}{Shared limitation}
    Both algorithms are \textbf{on-policy}: they cannot reuse data from earlier
    training phases, limiting sample efficiency relative to off-policy methods
    (DQN, SAC).
  \end{alertblock}
\end{frame}
